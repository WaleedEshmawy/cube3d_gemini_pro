Cube3D v4.0.0 - Function Logic & BreakdownThis document provides a detailed, step-by-step explanation of the User Interface and every JavaScript function inside the Cube3D application.Table of ContentsUI Button Descriptions3D Geometry & Rendering FunctionsState Management & Utility FunctionsAlgorithm / Solve FunctionsMovement & Animation ProcessorsInput Bindings & Mouse/Touch ControlsAutomated Testing ModuleGlobal Event Listeners & Chat ParserUI Button DescriptionsThe application features a control panel on the left side and a step-by-step solve tracker on the right side.Shuffle: Applies 20 random 90-degree slice rotations to scramble the cube completely.Reset: Instantly resets the cube to its pristine, solved canonical state (White top, Blue front, etc.) and clears progress.Solve: Automatically triggers the step-by-step solving algorithm from the current scrambled state. It utilizes the cube_solve() function to progress through Steps 1 to 6. It stops and alerts you if it encounters an unmapped configuration.Toggle Corners (Hidden by Default): Toggles the visual Arabic/Roman numeral overlays on target pieces. This button only appears once the solver algorithms have identified pieces.Save: Serializes the exact current 3D cube state into a text string and automatically downloads it to your computer as a .txt file.Load: Opens an OS file browser allowing you to select and load a previously saved .txt cube state into the canvas.Record: Toggles macro recording mode. When active, every manual slice rotation you make is logged. When you click it again to stop, it allows you to assign those recorded moves to a missing lookup table entry via the terminal.Test Loop: Initiates an infinite, automated testing cycle. It shuffles the cube, completely randomizes the 3D camera angle, and attempts to solve it autonomously. It tracks successful consecutive passes and stops immediately if a failure occurs.Get View: Prints the current raw X and Y camera angles to the terminal.Dump Tables: Exports all currently loaded algorithm sequence lookup tables (corners, edges, etc.) into a downloadable .txt file for backup.Back Move (Bottom Right): Reverses the last executed mathematical slice rotation. If recording mode is active, it dynamically prunes the recorded macro string.1. 3D Geometry & Rendering FunctionsgetSliceIndices(f, r, c)Takes a 2D face index f (0 to 5) and grid row/col r and c.Evaluates the face index to determine which 3D axis is fixed (e.g., Face 0 fixes y at 0).Maps the 2D row/col to the dynamic x and z axes respectively.Returns: An object {x, y, z} representing the array.rotate3D(x, y, z, ax, ay)Receives a 3D coordinate and camera angles (ax for pitch, ay for yaw).Applies a standard 2D rotation matrix around the Y-axis using ay (Yaw).Applies a second rotation matrix around the X-axis using ax (Pitch) on the new coordinates.Swaps/inverts the resulting axes specifically for proper isometric canvas projection.Returns: [x2, -z2, y2].project3D(x, y, z)Passes the raw 3D coordinate to rotate3D() to factor in the camera angles.Calculates a perspective scaling factor based on the resulting Z-depth and the global fov.Multiplies the rotated X and Y by the scaling factor and adds the canvas center point (cx, cy) to center the projection.Returns: The final 2D [px, py] canvas coordinates.rotateSlice3D(x, y, z, axis, angle)Takes a 3D coordinate, the axis to rotate around (x, y, or z), and the angle in radians.Checks which axis is being rotated.Applies a 2D rotation matrix to the other two coordinate axes using standard Math.cos and Math.sin formulas.Returns: The new rotated 3D coordinate.rotateSliceArray(axis, index, dir)Converts the directional move (1 or -1) into a 90-degree radian angle (PI/2).Creates a completely deep clone of the 3x3x3 cube3D data structure so mutations don't overlap during execution.Iterates over every single coordinate (x, y, z) from 0 to 2.Checks if the current coordinate belongs to the slice moving on the chosen axis.If in the slice, passes the coordinate to rotateSlice3D() to find out where this cubie physically ends up.Rounds the result to snap it to the new 0, 1, or 2 integer array index.Iterates over the cubie's color facings. If a color exists, applies rotateSlice3D to the face's normal vector to determine where that color is pointing after the rotation.Writes the reorganized cubie into the newly cloned cube array at its new coordinates.Overwrites the global cube3D array with the updated clone.draw(activeAnim)Clears the HTML5 canvas.Creates an empty projectedFaces array.Iterates over all 54 items in the global faces array.Checks if an animation is provided (activeAnim). If so, checks if the current face belongs to the moving slice and applies rotation data.Loops over the 4 vertices of the face, rotating them in 3D space if animating.Projects the 3D vertex to 2D coordinates using project3D() and calculates the average Z-depth of the 4 points.Looks up the current color of the facelet from the global cube3D array using the stored array mappings.Pushes the 2D polygon, color, and average depth to projectedFaces.Sorts projectedFaces based on depth (Painter's Algorithm: back to front).Caches the array into lastProjectedFaces for mouse picking later.Iterates over sorted faces, drawing the filled polygon path and stroking the black borders.If corner indications are enabled, calculates the 2D center point of the facelet.If the facelet belongs to an identified target corner or side, draws its respective number or Roman numeral directly onto the polygon using canvas text. Overlays markers like o for orientations.Projects, sorts, and draws the small Red/Green/Blue 3D axis indicator in the top right corner.2. State Management & Utility FunctionsupdatePassedCountUI()Grabs the passedTestCount HTML element.Overwrites its text content with the current value of the global passedTestCount variable.updateStepUI(maxStep)Loops through step identifiers (1 to 6).If the iterator is less than or equal to maxStep, removes the disabled CSS class from the right-panel buttons.If greater, adds the disabled CSS class.resetStepProgress()Resets currentMaxStep to 1 and calls updateStepUI(1).Hides the HTML elements that display the target face and side face color palettes.Clears internal identification arrays and resets state flags.Hides the "Toggle Corners" button and strips gradient background progress visually from all step buttons.checkSolved()Iterates over the top layer and verifies every cubie's 'top' color matches the top center piece.Repeats this exhaustive matching iteration for the bottom, front, back, left, and right faces against their respective center pieces.Returns: true if every single facelet aligns; otherwise returns false.updateSolvedStatus()Evaluates the boolean output of checkSolved().If true, adds the active CSS class to the solved indicator, empties the move history, calls resetStepProgress(), and forces max step to 0 to disable buttons.If false, removes the active class and ensures Step 1 is activated if progress was reset.resetCube()Guards against firing while animations are currently running.Re-initializes the global cube3D array from scratch, manually assigning default colors (Blue front, White top, etc.).Resets camera angles to standard isometric defaults.Clears history, resets steps, runs updateSolvedStatus(), and triggers draw() to re-render.logMoveToTerminal(axis, index, dir, steps)Maps semantic axes (x, y, z) to numeric values (0, 1, 2) and rotation directions (1, -1) to bit strings (0, 1).Concatenates the components into standard shorthand format (e.g., "02:11").Calls addLog() to print the move into the visible UI terminal.Checks if the isRecording flag is active. If so, pushes the shorthand string into the recordedMoves array.shuffleCube()Clears the solved indicator and resets the step progress pipeline.Runs a loop 20 times.For each iteration, randomly selects an axis, a slice index (0, 1, 2), and a direction.Constructs an animation object for the move, sets a rapid speed, pushes it to animationQueue, logs to terminal, and kicks off requestAnimationFrame(animateTurn).serializeCube()Extracts the 6 core center colors from the 3D data structure to generate a shorthand layout prefix (e.g., "WBYROG").Iterates over every x, y, z coordinate in the cube3D array.Converts existing color facelets to character codes (T, O, F, B, R, L) and pairs them with their coordinate block.Returns: A 55-chunk, space-separated string containing the entire exact physical state.deserializeCube(str)Splits the incoming string argument by whitespace and validates chunk length.Generates a blank, 3x3x3 cube3D null array.Loops over the parts of the string in pairs (XYZ chunk and Colors chunk).Parses the characters, maps the letters back to numerical color IDs, and writes them directly into the specific properties of the targeted cubie.Overwrites the global cube3D array and returns true.saveCube(), loadCube(), dumpTablesToFile()File I/O handlers for converting internal string/dictionary state to dummy Blob URLs for automatic .txt file downloading, and <input type="file"> reading.3. Algorithm / Solve FunctionsalignCubeToSelection()Runs matrix calculations simulating where the camera is looking based on angleX and angleY.Computes the mathematical dot product of the camera vectors against standard face normal vectors to deduce which face is pointing most "UP" and "FRONT".Executes sequence of applyWholeCube logic statements to physically rotate the entire 3D array data so the viewed orientation aligns with the data index axes.Resets camera angles back to default seamlessly.executeStep1()Bails out if an animation is currently executing.Runs alignCubeToSelection().Looks up the canonical front color corresponding to the current top center color.Rotates the entire cube along the Z-axis until the front center matches the canonical requirement.Updates UI color squares, upgrades to Step 2, snaps camera to 0.625 angle, and renders.top_corner_identify(printToTerminal)Retrieves center colors and builds target objects for the 4 top corners.Exhaustively loops through the 8 physical corners of the array.Extracts present colors. If a cubie matches a target, computes orientation and 'solved' status.Pushes tracking object to topCornersData, recalculates visual progress bar percentage.If requested, logs unsolved edge paths to terminal. Updates canvas overlay numerals.top_corner_solve(corner)Constructs a string key (e.g., c1_200:1) representing the corner's exact location and twisting orientation.Looks up the sequence from CORNER_SOLUTIONS.Dispatches to executeShorthandSequence() if found, logs failure if missing.executeStep2()Locks isStep2Running.First Click: Runs identifying logic silently, turns on UI overlays.Subsequent Clicks: Re-identifies state silently, grabs first unsolved corner, and triggers top_corner_solve().Hooks an anonymous callback to check if all corners are finally solved post-animation, and upgrades to Step 3.top_side_identify, top_side_solve, top_side_check, executeStep3()Mirrors the logic of the Step 2 corners, but analyzes the 2-color edge pieces across the TOP_SIDE_SOLUTIONS table to assemble the top cross.edge_side_identify, edge_side_solve, mid_edge_check, executeStep4()Mirrors Step 3, but targets the 4 middle-layer vertical edge pieces using the EDGE_SIDE_SOLUTION table. Ensures alignment strictly to the canonical Front/Back axes.bottom_corner_identifyMeasures the 4 bottom-layer corners. Critically, it tracks isLocationSolved (XYZ match) vs isOrientSolved (Twist match) independently rather than one binary solved value.bottom_corner_solve_location(wrongLocations)If 3 or 4 corners are out of place, applies a baseline bottom layer rotation ("02:01") to alter positions geometrically.If 1 or 2 corners are out of place, computes a location-based key (e.g., bcl1_000) and executes sequence from BOTTOM_CORNER_LOC_SOLUTION.bottom_corner_solve_orientation(wrongOrientation)Determines orientation pattern by grouping corners by the horizontal face their 'bottom' color points towards.Priority 1: Finds if two corners face the identical side (triggering BCO2).Priority 2: Locates one solved corner and an adjacent corner pointing toward an interconnected shared face (triggering BCO1).Executes sequence from BOTTOM_CORNER_ORIENT_SOLUTION.executeStep5()Enforces Step 4 completion. Changes view to the bottom face.Evaluates location array first. If unlocated, fires bottom_corner_solve_location().If locations are perfect but twisted, fires bottom_corner_solve_orientation().Hooks recursive callback to evaluate completion and advance to Step 6.bottom_side_identify(printToTerminal)Isolates the 4 bottom edges and records location/orientation properties.bottom_side_solve_location()Checks if exactly 1 bottom side is correctly located. If so, retrieves the side piece resting in its relative left-adjacent slot to compute an origin-destination key (e.g., bsl_1_4).If 0 edges are correctly located, it picks a "BSL ANY" sequence out of BOTTOM_SIDE_LOC_SOLUTION array zero-index to safely force a permutation cycle shift.Executes the retrieved sequence or traps a missing CLI command.bottom_side_solve_orientation(wrongOrientation)Collects the identifiers of pieces resting correctly but flipped.Sorts identifiers into a concatenated string key (e.g., bso_23).Evaluates BOTTOM_SIDE_ORIENT_SOLUTION and dispatches shorthand rotation array.executeStep6()Locks pipeline, alters camera angle, evaluates location followed by orientation for the final 4 edges. On final successful pass, forces updateSolvedStatus() to fully complete the app solve.4. Movement & Animation ProcessorsrotateLayer(axis, layer, dirString, steps)Validates axis (x/y/z) and layer index.Maps semantic directions to integers (1 or -1).Loops steps times, pushing a move instruction object to animationQueue.Kicks off requestAnimationFrame(animateTurn) if not animating.executeShorthandSequence(val)Uses regular expression /^[012][012]:[01][12]$/ to validate formatting.Splits strings into sequences.Iterates over valid moves, parses the regex matches into arguments, and routes them to rotateLayer().animateTurn()If queue is empty, flips isAnimating false, triggers draw(), fires onAnimationComplete closures, and halts.Extracts first move object and adds speed to progress.If progress > 90-degrees (Math.PI / 2), finalizes rotation in data structure via rotateSliceArray(), shifts object off queue, and recurvsively loops to next turn.If progress < 90-degrees, calls draw(anim) to render the transitional twist and requests next animation frame.5. Input Bindings & Mouse/Touch ControlshandleInteractionStart(x, y)Blocks input if animating. Stores starting coordinates.Iterates backwards through lastProjectedFaces (front to back). Uses pointInPolygon to see if user clicked inside a rendered face.Caches target to dragStartFace or enables isGlobalDragging if missed.handleInteractionMove(x, y)If isGlobalDragging, increments global angleX/angleY camera values and calls draw().If dragging a face:Computes drag distance.Looks up valid 3D axes a specific face is legally allowed to turn on.Simulates micro-rotations for each axis/direction and computes mathematical dot product comparing user's swipe vector against the simulated 2D line.If the best dot product > 0.5, packages the resulting optimal move into animationQueue and fires animateTurn().6. Automated Testing ModulewaitForAnimation() [Async]Initiates an asynchronous while-loop. Forces the thread to sleep for 100ms chunks using JavaScript Promises if animations are currently processing, blocking the surrounding test routine until the physical UI completely settles.cube_solve() [Async]Externalized master solve loop.Sequentially executes executeStep1() through executeStep6().Between every step execution, it yields to waitForAnimation().Utilizes for loops to limit the max number of clicks/attempts allowed per step (e.g. up to 16 times for Step 5).If the executeStepX module returns false (indicating a missing entry in a lookup table), or if the loop caps out without incrementing currentMaxStep, it immediately returns false to abort the solve gracefully.Returns: true if checkSolved() verifies absolute completion.runAutomatedTest() [Async]Infinite while-loop contingent on the isTesting flag.Triggers shuffleCube() and yields.Completely randomizes floating-point angleX and angleY coordinate views, redraws, and yields.Awaits the boolean result of cube_solve().If true, increments global passedTestCount, updates the UI, yields, and restarts the shuffle loop.If false, traps the error, breaks out of the while loop, and disables the testing UI flags.7.
